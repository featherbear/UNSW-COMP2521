It is fairly easy to write a function to determine whether an array of integers is sorted. A trickier problem is to determine whether the sort algorithm that produced a sorted array is stable.

Assume that the array is composed of Items with two fields, as in:

typedef struct { int a; int b; } Item;

Assume also that the array has been sorted on the Item.a field.

Given the final sorted array alone, you cannot determine whether the sort was stable. We also assume that the sorting client keeps a copy of the original array, which is available for checking.

Given the above, write a function that takes the original array, the sorted array, and determines whether the sort was stable. Use this prototype:

bool stable_sort_p (Item *unsorted, Item *sorted, int lo, int hi);

This solution is not particularly efficient. If you come up with a better one, let me know. In particular, you could save a lot of checking if you remembered each key value that had already been checked, and skipped it when you see it again:

bool stable_sort_p (Item *unsorted, Item *sorted, int lo, int hi)
{
	for (int i = lo; i <= hi; i++) {
		int key = sorted[i].a;
		// setting j = i means that we "find"
		// the current key in position i first
		int j = i, k = 0;
		while (j <= hi and k <= hi) {
			// find next pair of items with (item.a == key)
			for (/* j */; j <= hi; j++)
				if (sorted[j].a == key) break;
			for (/* k */; k <= hi; k++)
				if (unsorted[k].a == key) break;
			// check that they have the same item.b
			if (j <= hi && k <= hi) {
				if (sorted[j].b != sorted[k].b)
					return false;
			}
		}
	}
	return true;
}

Consider the implementations of the following sorting functions.

/// Bubble-sort.
void sort_bubble (int items[], size_t n)
{
	for (size_t i = n - 1; i > 0; i--)
		for (size_t j = 1; j <= i; j++)
			if (items[j] < items[j - 1])
				swap_idx (items, j, j - 1);
}

/// Bubble-sort with early exit.
void sort_bubble_ee (int items[], size_t n)
{
	bool no_swaps = false;
	for (size_t i = n - 1; i > 0 && !no_swaps; i--) {
		no_swaps = true;
		for (size_t j = 1; j <= i; j++)
			if (items[j] < items[j - 1]) {
				no_swaps = false;
				swap_idx (items, j, j - 1);
			}
	}
}

/// Selection sort.
void sort_selection (int items[], size_t n)
{
	for (size_t i = 0; i < n; i++) {
		size_t low = i;
		for (size_t j = i + 1; j < n; j++)
			if (items[j] < items[low])
				low = j;
		swap_idx (items, i, low);
	}
}

Which implementations are adaptive?

Only the bubble sort with early exit, which is instead of for already-sorted data. After one iteration, no swaps are performed, so the program ends.

Which implementations are stable?

All but selection sort are stable. This implementaiton of selection sort sacrifices stability to be in-place. Elements can be swapped from one end to another, jumping over possible duplicates. To prove instability you can simply find a particular instance of data where duplicates would end up out of the original order.

(b)Link, and you’ll miss it

Write a version of selection sort that builds a new sorted list from an original unsorted linked list. The original list should not be modified during the sorting.

Here’s a linked list definition –

struct node {
	int item;
	node *next;
};

– and here’s a prototype:

node *list_sort_selection (node *ls);

Is your implementation stable?

Since we cannot modify the original list, the first thing to do would be to make a copy of the original list. Keeping the spirit of array-based selection sort, each iteration would:

  • find largest element remaining in the copy,
  • remove that element from the copy, and
  • insert it at the front of the new list.
static node *node_new (int item);
static node *node_new2 (int item, node *next);
static node *copy (node *ls);

node *list_sort_selection (node *ls)
{
	node *old = copy (ls);
	node *res = NULL;  // result list

	while (old != NULL) {
		// find largest
		node *max = old;
		for (node *cur = old; cur != NULL; cur = cur->next)
			if (cur->value > max->value)
				max = cur;

		// unlink from old list
		node *prev = NULL;
		for (node *cur = old; cur != max; cur = cur->next)
			prev = cur;
		if (prev == NULL)
			old = max->next;
		else
			prev->next = max->next;

		// push at start of result list
		max->next = res;
		res = max;
	}

	return res;
}

static node *copy (node *ls)
{
	if (ls == NULL) return NULL;
	return node_new2 (ls->value, copy (ls->next));
}

static node *node_new2 (int item, node *next)
{
	node *new = malloc (sizeof *new);
	if (new == NULL) err (EX_OSERR, "couldn't allocate node");
	(*new) = (node) { .item = item, .next = next };
	return new;
}

Yes, it is stable.

What the Shell!

Consider the following Shell sort algorithm, based on Sedgewick’s implementation:

void sort_shell (int items[], size_t n)
{
	// Calculate starting `h'-value
	size_t h;
	for (h = 1; h <= (n - 1) / 9; h = (3 * h) + 1)
		printf("%zu ", h);
	puts ("");

	for (/* h */; h > 0; h /= 3) {
		// when `h' = 1, this is an insertion sort.
		for (size_t i = h; i < n; i++) {
			int key = items[i];
			size_t j = i;
			for (/* j */; j >= h && key < items[j - h]; j -= h)
				items[j] = items[j - h];
			items[j] = key;
		}
	}
}
